Автоматизация теста к API

Автоматизация сценария: Клиент создает заказ. Проверяется, что по треку заказа можно получить данные о заказе.

Шаги автотеста: Выполнить запрос на создание заказа. Сохранить номер трека заказа. Выполнить запрос на получения заказа по треку заказа. Проверить, что код ответа равен 200.
## Описание сценария автотеста
1. Выполняется POST-запрос на создание нового заказа (`/api/v1/orders`).
2. Из тела ответа извлекается и сохраняется трек-номер (`track`).
3. Выполняется GET-запрос на получение данных заказа по сохраненному треку (`/api/v1/orders/track?t=...`).
4. Проверяется, что система возвращает статус-код `200 OK`.

## Структура проекта
* `configuration.py` — конфигурационный файл с базовым URL стенда и путями к ручкам (API endpoints).
* `data.py` — тестовые данные (тело запроса создания заказа и заголовки).
* `sender_stand_request.py` — функции отправки HTTP-запросов (POST и GET) с использованием библиотеки `requests`.
* `test_get_order.py` — тестовый скрипт со сценарием проверки и утверждением (assert).
